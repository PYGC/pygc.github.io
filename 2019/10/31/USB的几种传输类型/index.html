USB的几种传输类型

2019-10-31

USB支持四种传输类型：

  • Control（控制）
  • Bulk（批量）
  • Interrupt（中断）
  • Isochronous（同步)

在控制传输中，主机将已定义的请求发送到设备。在设备连接时，主机使用控制传输从设备请求一系列称为描述符的数据结构。描述符提供有关设备功能的信息，并帮助主机确定要分配给设备的驱动程序。类规范或供应商也可以定义请求。

控制传输最多分为三个阶段：

  • Setup（设置）
  • Data（数据，可选）
  • Status（状态）

Setup 阶段包含该请求。 如果存在，则 Data 阶段将根据请求包含来自主机或设备的数据。Status 阶段包含有关传输成功的信息。 在控制读取传输中，设备在数据阶段发送数据。 在控制写传输中，主机在 Data 阶段发送数据，或者不存在 Data 阶段。

其他传输类型没有定义阶段。 相反，高级软件定义了如何解释原始数据。

在原本空闲的总线上，Bulk 传输是最快的，但没有保证的时间。 打印机和驱动器使用 Bulk 传输。

Interrupt 传输可确保最大延迟或两次尝试之间的时间间隔。 鼠标和键盘使用 Interrupt 传输。

Isochronous 传输保证了时序，但没有纠错。 流音频和视频使用 Isochronous 传输。

USB 通信分为两大类：

  • 有助于识别和配置设备的通信
  • 实现设备用途的通信

在枚举期间，主机了解设备并请求配置。 枚举完成后，主机可以根据需要发送和请求数据以实现设备的用途。

枚举期间，设备的固件会响应主机发出的一系列标准请求。设备必须解码请求，返回请求的信息，并采取其他措施执行请求。

Windows 和其他操作系统无需执行应用程序编程即可执行枚举。在Windows下，设备首次连接到系统时，即插即用（PnP）管理器必须找到一个INF文件，该文件标识一个或多个要分配给该设备的驱动程序文件的名称和位置。如果所需文件可用且固件正常运行，则枚举过程通常对用户不可见。

主机枚举设备并分配和加载设备驱动程序后，即可开始应用程序通信。在主机上，应用程序可以使用 Windows API 函数或其他软件组件来读取和写入设备。在设备上，传输数据通常需要将数据放置在端点的发送缓冲区中发送或从端点的接收缓冲区中检索接收到的数据，并在完成事务后确保端点准备好进行另一笔事务。大多数设备还需要固件支持才能处理错误和其他事件。
